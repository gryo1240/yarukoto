📱⇄📱 ふたりのスマホを同期する
かんたん設定ガイド

やることは「Googleの無料の保管場所を1つ作って、その住所をアプリに教える」だけ。1回やれば、3つのアプリ(やること・せんたく・そうじ)全部で使えます。スマホでもできますが、パソコンのほうがラクです。

じゅんび

用意するもの

Googleアカウント(〜@gmail.com のメールアドレス)とそのパスワードだけ。ふたり分は要りません。どちらか1人のアカウントでOKです。

STEP 1

Firebaseのページを開いてログイン

  1. console.firebase.google.com を開く(タップで開きます)
  2. Googleのログイン画面が出たら、いつものGmailアドレスとパスワードでログイン

💡 「Firebase」はGoogleがやっている無料のデータ保管サービスの名前です。あやしいものではありません。

STEP 2

「プロジェクト」を1つ作る(=自分専用の保管場所)

  1. プロジェクトを作成(または「開始する」)という大きなボタンを押す
  2. 名前を聞かれたら kazoku と入力(何でもOK)→ 続行
  3. 「Gemini」や「AI」の案内が出たら、スイッチはオフのままで 続行
  4. 「Googleアナリティクス」のスイッチをオフにする → プロジェクトを作成
  5. くるくる回るのを待って 続行 を押す
STEP 3

データベース(保管箱)を作る

  1. 画面の左のメニューから 構築 を押して、中の Realtime Database を押す
    (スマホで左メニューが見えないときは、左上の「≡」を押すと出ます)
  2. データベースを作成 を押す
  3. 「ロケーション」はそのままで 次へ
  4. テストモードで開始 を選んで 有効にする
STEP 4

カギのかけ方を貼り替える(コピペするだけ)

  1. データベースの画面の上のほうにある ルール というタブを押す
  2. 下のボタンでルールをコピーする
  3. 画面に元から書いてある文字を全部消して、コピーしたものを貼り付ける
  4. 公開 ボタンを押す(「公開されました」と出ればOK)
{ "rules": {
  "rooms":  { "$rid": { ".read": true, ".write": true } },
  "closet": { "$rid": { ".read": true, ".write": true } },
  "souji":  { "$rid": { ".read": true, ".write": true } }
} }

💡 これは「アプリの招待コードを知っている家族だけが読み書きできる」というカギの設定です。

STEP 5

「config」(保管場所の住所メモ)をコピーする

  1. 画面の左上にある ⚙(歯車マーク) → プロジェクトの設定 を押す
  2. 画面を下にスクロールして「マイアプリ」のところにある </>(ウェブという丸いボタン)を押す
  3. 「アプリのニックネーム」に kazoku と入れて アプリを登録(チェックボックスは触らなくてOK)
  4. すると const firebaseConfig = { apiKey: "…" … }; という文字のかたまりが出ます。この { から } までを全部コピー(かたまりの右上にコピーボタンが出ることもあります)

⚠ このメモ(config)と招待コードは「家のカギ」と同じです。家族以外には送らないでください。

STEP 6

アプリに貼り付ける(1台目のスマホ)

  1. 「きょうのやること」アプリを開く → ⚙設定 タブ
  2. 「夫婦で同期」のところの入力欄に、さっきコピーしたものをそのまま貼り付け
  3. 同期を開始 を押す →「同期を開始しました」と出ます
  4. 📋 招待コードをコピー を押して、LINEなどで相手に送る
STEP 7

相手のスマホで招待コードを貼る(2台目)

  1. 相手のスマホでも「きょうのやること」アプリを開く → ⚙設定 タブ
  2. 「もう1台のスマホはこちら」の欄に、LINEで届いた招待コードを貼り付け
  3. 招待コードで参加 を押す

🎉 これで完成! 片方がチェックすると、数秒でもう片方のスマホにも反映されます。画面右上の丸が緑なら同期中です。

💡 「せんたくタグ」「そうじナビ」も同期したいときは、各アプリの設定タブに同じconfigを貼って「同期を開始」→ 出てきた招待コードを相手に送る、だけです(STEP1〜5のやり直しは不要)。

こまったときは

Q. 画面が英語で出てくる
A. ページの一番下に言語選択があれば「日本語」に。無くてもボタンの場所は同じです(Create project=プロジェクトを作成、Build=構築、Rules=ルール、Publish=公開)。
Q. 「書き込みに失敗しました」と出る
A. STEP 4のルールの貼り替えがうまくいっていません。もう一度「ルール」タブを開いて、全部消してから貼り直して「公開」。
Q. configに databaseURL が無いと言われた
A. STEP 3(データベース作成)を飛ばしています。STEP 3をやってから、Realtime Databaseの画面の上に出ている https://〜firebasedatabase.app という文字をコピーして、アプリの追加欄に貼ってください。
Q. お金はかかる?
A. かかりません。家族のメモ程度のデータ量では無料枠の1%も使いません。クレジットカード登録も不要です。
